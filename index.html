Robot IoT Telemetry & Cloud Integration

This project focuses on bridging the gap between autonomous robotics and remote monitoring. By utilizing IoT (Internet of Things) protocols, I capture high-frequency data from robot sensors and transmit it to the cloud for real-time analysis and historical logging.

The Goal: To create a universal telemetry hub that allows any robot—regardless of its environment—to report its status, sensor readings, and health metrics to a centralized dashboard via ThingSpeak.

How the IoT Stack Works

The core of this system is designed to be hardware-agnostic, though I primarily utilize the Heltec WiFi Kit 8 (ESP8266) for its integrated OLED display and low power consumption.

  • Data Capture: The robot gathers inputs such as distance (Laser ToF), temperature, and system voltage.
  • Wireless Transmission: Data is packaged into HTTP GET/POST requests and sent via local WiFi.
  • Cloud Visualization: ThingSpeak receives the data, providing live gauges, charts, and public/private API access for further processing.

The Proof of Concept: PondGuard

Success Story: This system was first deployed as "PondGuard." It successfully utilized laser sensors and thermal probes to monitor a pond's health. The ability to see water levels on a smartphone from anywhere in the world proved the reliability of the ThingSpeak + ESP8266 stack.

Lessons learned from the PondGuard project—such as waterproof housing and stable WiFi reconnect logic—are now being integrated into my mobile robotic platforms to monitor battery levels and navigation accuracy.

Technical Specification

  • Controller: Heltec WiFi Kit 8 (ESP8266) / ESP32.
  • Sensors: Time-of-Flight (ToF) Lasers, DS18B20 Temperature Probes, IMUs.
  • Platform: ThingSpeak IoT Cloud.
  • Codebase: C++ / Arduino with asynchronous WiFi libraries.
