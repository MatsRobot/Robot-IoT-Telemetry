System Architecture Overview

Technical Framework Features

This project utilizes an edge-to-cloud pipeline to ensure robot health and environmental data persistence:

  • Cloud-Native Architecture: Direct integration with ThingSpeak for live graphing and historical data storage.
  • Precision ToF Sensing: Uses VL53L0X Laser sensors for millimetre-accurate obstacle or level detection.
  • Multi-Node Thermal Monitoring: Implements the 1-Wire protocol for daisy-chained DS18B20 temperature probes.
  • Remote Alarming: Logic triggers (ThingSpeak React / IFTTT) for critical threshold notifications.

Operational Logic: The "PondGuard" Implementation

The system follows a synchronized four-step cycle to ensure reliable telemetry:

  1. Sensing: The hub polls the Laser ToF sensor and multi-point temperature probes.
  2. Processing: The ESP8266 calculates deltas and formats the HTTP payload for transmission.
  3. Cloud Sync: Data is pushed via WiFi to the ThingSpeak REST API.
  4. Action: If parameters exceed safe limits (e.g., low battery or low level), an automated alarm is triggered.
